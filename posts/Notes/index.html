    C/C++

    std::shared_ptr #

    Consider the following codes below:

    struct Base {
    virtual ~Base() {}
};

struct Derived : public Base {
    virtual ~Derived() {}
};

int do_something(const std::shared_ptr<Base>& a) {
    return 0;
}

void wrapper(const std::shared_ptr<Derived>& a) {
    int t = do_something(a);
}

int main() {
    std::shared_ptr<Derived> t = std::make_shared<Derived>();
    wrapper(t);
}

    Everythings seems OK but if we use the annotation [[clang::lifetimebound]] from clang, we could get the following warning:

    warning: temporary whose address is used as value of local variable ’t’ will be destroyed at the end of the full-expression [-Wdangling]

    Something about hash #

    Some hash functions distribute information evenly across all their bits — any change to the input causes an avalanche of changes to the hash code. Ideally, any one-bit change in the input is expected to change about half the bits of the output. These hash functions can be mapped onto a power-of-two range by just zeroing the top bits. For example, we might map hash codes across 32 chunks by zeroing all but the bottom five bits.
